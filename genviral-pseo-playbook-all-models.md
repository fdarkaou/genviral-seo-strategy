### Model 1: UGC Gallery Pages (photo.ai model)
**Scale: 10k → unlimited | Traffic: proportional to user base**

**How it works:**
Every AI generation by a user becomes a public indexed page. The prompt becomes the URL slug. The output IS the content. No manual content creation.

```
photoai.com/woman-with-coquette-hair-half-up-with-ribbon-27358990
photoai.com/corporate-headshots-in-dark-academia-office-27351234
```

28M photos generated = potentially 28M indexable pages. They keep only public ones in sitemap (~5,500 in sitemap but much more indexed via crawl).

**Traffic source:** Hyper-specific long-tail. "Woman with coquette hair half up with ribbon" is 0 search volume on its own, but 600k pages × 3 long-tail hits each = 1.8M/month.

**GenViral equivalent:**
```
genviral.io/gallery/fitness-tiktok-slideshow-morning-motivation-[id]
genviral.io/gallery/restaurant-instagram-reel-menu-reveal-[id]
genviral.io/gallery/beauty-youtube-short-skincare-routine-[id]
```

Every generated slideshow/video → public page with:
- The slides rendered as preview images
- The niche + platform + content type tags
- AI-generated caption/hook/hashtag for that niche (from the generation context)
- "Remix this template →" CTA into the Genviral editor
- "See more fitness TikTok slideshows →" links to related gallery

**You already have 3,296 users. If 30% make content public = ~10k pages from day one. At 10k users = 100k pages.**

### Model 2: Function × Parameter Matrix (HeyGen + Kapwing model)
**Scale: 100 → 10,000 pages | Traffic: high per page**

**How it works:**
Take one high-value function. Multiply it across every valid parameter combination.

**HeyGen** built `/translate` → massive traffic from "translate [language] video" searches:
- "translate" = 45.5M searches/month — pos. 59
- "spanish into english" = 9.1M — pos. 72
- "translate english to spanish" = 5M — pos. 56

**Kapwing** built `/tools/translate/audio/spanish-to-english`, `/tools/translate/audio/French-to-english`, etc.:
- 9.1M monthly searches for "spanish to english" — pos. 21-72
- Each language pair = its own page

**The math:** 20 source languages × 20 target languages = 400 pages, capturing combined hundreds of millions of monthly searches.

**GenViral already has this model — it's your download tools.**
You have 21 platform downloaders. `tools/download/tiktok` is sitting on 246k searches/month (KD 36). `tools/download/instagram` on 368k. You have 21 of these. That's the matrix.

### Model 4: Template Gallery (Canva model)
**Scale: 1,000 → 10M pages | Traffic: 88M/month for Canva**

**How it works:**
Every design/template gets its own public URL. Categories, sub-categories, and individual template pages create a massive hierarchy.

**Canva's structure:**
```
canva.com/instagram-stories/templates/                    → 30M/mo searches
canva.com/[niche]/templates/[category]/                   → hundreds of thousands of pages
canva.com/templates/[specific-template-name]-[id]/        → individual template page
```

Canva has 3.3M ranking keywords and 88M monthly US traffic. Most of it flows through template pages.

**GenViral equivalent:**
```
genviral.io/templates/                                    Hub
genviral.io/templates/tiktok/                             Platform hub
genviral.io/templates/tiktok/slideshow/                   Content type hub
genviral.io/templates/tiktok/slideshow/fitness/           Niche page
genviral.io/templates/tiktok/slideshow/fitness/yoga/      Sub-niche page
genviral.io/templates/[template-name]-[id]/               Individual template
```

**The individual template page** is where scale explodes. If GenViral has 1,000 user-created or AI-generated templates across all niches, each gets an indexed page. If your platform grows to 100,000 templates = 100,000 pages, all auto-generated, all with unique visual content (the template preview).

Canva's competitive moat is that their templates ARE the content. Nobody can copy a Canva template page because the template visual is unique. Same applies to GenViral's slideshow templates.
